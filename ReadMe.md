# Terraform: развёртывание Plex Media Server в Yandex Cloud

Автоматизированное создание инфраструктуры в Yandex Cloud с помощью Terraform: виртуальная сеть, подсеть и виртуальная машина с предустановленным Plex Media Server.

## Что будет создано

При применении конфигурации Terraform развернёт следующие ресурсы в Yandex Cloud:

* **Виртуальная сеть (VPC)**: `network_1` с подсетью `10.2.0.0/24` в выбранной зоне доступности.
* **Виртуальная машина (ВМ)**: инстанс `terraform-vm` на платформе `standard-v3` (2 vCPU, 2 GB RAM).
* **Операционная система**: Ubuntu 24.04 LTS.
* **Plex Media Server**: автоматически устанавливается при первом запуске ВМ через `cloud-init`.

## Требования

Перед началом работы убедитесь, что у вас есть:

1. Установленный **Terraform версии не ниже 0.13**.
2. Аккаунт в **Yandex Cloud** с достаточным балансом и правами для создания ресурсов.
3. Настроенная **аутентификация** в Yandex Cloud (через переменные окружения или сервисный аккаунт).

## Переменные конфигурации

Для работы конфигурации необходимо задать следующие переменные:

| Переменная | Описание | Пример значения |
|----------|----------|---------------|
| `cloud_id` | ID облака Yandex Cloud | `b1g1234567890abcdef` |
| `folder_id` | ID каталога, в котором будут создаваться ресурсы | `b1g9876543210fedcba` |
| `zone` | Зона доступности | `ru-central1-b` |
| `ssh_public_key` | Публичный SSH‑ключ для доступа к ВМ (формат: `ssh-rsa AAAA...` или `ssh-ed25519 ...`) | `ssh-rsa AAAAB3NzaC1yc2E... user@host` |

### Способы задания переменных

* Через файл `terraform.tfvars` (рекомендуемый способ).
* С помощью флага `-var` при запуске команд Terraform.
* Через переменные окружения вида `TF_VAR_<имя_переменной>` (например, `TF_VAR_cloud_id`).

## Инструкция по использованию

### Шаг 1. Подготовка переменных

1. Скопируйте пример файла с переменными:

   ```bash
   cp terraform.tfvars.example terraform.tfvars
2. Отредактируйте terraform.tfvars в любом текстовом редакторе и заполните значения:
|---------------------------------------------------------|
| cloud_id      = "b1g1234567890abcdef"
| folder_id   = "b1g9876543210fedcba"
| zone        = "ru-central1-b"
| ssh_public_key = "ssh-rsa AAAAB3NzaC1yc2E... user@host"

'Важно: не коммитьте terraform.tfvars в репозиторий. Добавьте его в .gitignore:

terraform.tfvars
*.tfstate
*.tfstate.backup
.terraform/

### Шаг 2. Инициализация Terraform
Выполните команду для загрузки провайдера Yandex Cloud и подготовки рабочей области:
* ** terraform init
* Ожидаемый результат:

В выводе появится сообщение Terraform has been successfully initialized!.

Создастся папка .terraform/ с зависимостями.

Шаг 3. Просмотр плана развёртывания
Проверьте, какие ресурсы будут созданы, без фактического применения изменений:

bash
terraform plan
Что проверить в выводе:

Список ресурсов с префиксом + (будут созданы).

Значения переменных, подставленные из terraform.tfvars.

Отсутствии ошибок валидации (Error: ...).

Шаг 4. Применение конфигурации
Запустите создание инфраструктуры:

bash
terraform apply
Система запросит подтверждение — введите yes и нажмите Enter.

Процесс развёртывания (2–5 минут):

Создание VPC‑сети и подсети.

Создание виртуальной машины terraform-vm.

Автоматическая установка Plex через cloud-init.

Ожидаемый вывод (фрагмент):

Apply complete! Resources: 3 added, 0 changed, 0 destroyed.

Outputs:

ip = "198.51.100.1"
Запишите или скопируйте значение ip — оно понадобится для доступа к Plex.

Шаг 5. Проверка и настройка Plex Media Server
Подключитесь к ВМ по SSH:

bash
ssh ubuntu@<IP-адрес-из-output>
Проверьте статус службы Plex:

bash
sudo systemctl status plexmediaserver
Убедитесь, что статус — active (running).

Проверьте логи установки:

bash
sudo cat /var/log/plex-install.log
В логе должны быть строки об успешной установке и запуске службы.

Откройте веб‑интерфейс Plex в браузере:

http://<IP-адрес-ВМ>:32400/web
Настройте Plex:

Войдите в учётную запись Plex или создайте новую.

Создайте библиотеку фильмов и укажите путь /media/movies как источник контента.

Добавьте медиафайлы: скопируйте видеофайлы в каталог /media/movies на ВМ (например, через scp или монтирование диска). Plex автоматически обновит библиотеку.

Полезные команды
Команда	Назначение
terraform plan -var="zone=ru-central1-a"	План с временной переменной (не меняет terraform.tfvars)
terraform show	Показать текущие созданные ресурсы и их атрибуты
terraform state list	Список ресурсов в состоянии Terraform
terraform output	Показать выходные значения (например, ip)
terraform destroy	Удалить все созданные ресурсы. Подтвердите yes для выполнения.
