# Terraform Examples — Yandex Cloud
## Пример конфигурации Terraform для развёртывания инфраструктуры в Yandex Cloud
Конфигурация позволяет развернуть инфраструктуру в Yandex Cloud, включающую виртуальную сеть, подсеть и виртуальную машину (ВМ) с автоматической установкой Plex Media Server.

## Что будет создано
В результате выполнения конфигурации будут развёрнуты следующие ресурсы:

- Виртуальная сеть (VPC): сеть с именем network_1 и подсетью с диапазоном IP‑адресов 10.2.0.0/24 в выбранной зоне доступности.

- Виртуальная машина: инстанс с именем terraform-vm, размещённый на платформе standard-v3 с характеристиками:

  2 виртуальных процессора (vCPU);

  2 ГБ оперативной памяти (RAM).

- Операционная система: Ubuntu 24.04 LTS.

- Plex Media Server: программное обеспечение устанавливается автоматически при первом запуске ВМ с использованием механизма cloud‑init.

## Требования к окружению

Для успешного выполнения конфигурации необходимо обеспечить выполнение следующих требований:
1) Установленный Terraform версии не ниже 0.13 (скачать можно на официальном сайте: ).
2) Наличие активного аккаунта Yandex Cloud.
3) Настроенная аутентификация в Yandex Cloud. Поддерживаются следующие способы:
  - через переменные окружения;
  - с использованием сервисного аккаунта (подробнее: ).

## Переменные конфигурации
Для настройки параметров развёртывания используются следующие переменные:

| Переменная       | Описание                    |
|------------------|-----------------------------|
| `cloud_id`       | Идентификатор облака в Yandex Cloud |
| `folder_id`      | Идентификатор каталога в Yandex Cloud |
| `zone`           | Зона доступности (например, `ru-central1-b`) |
| `ssh_public_key` | Публичный SSH‑ключ для доступа к виртуальной машине. Должен быть в формате: ssh-rsa AAAA... или ssh-ed25519 ... |

## Способы задания переменных
Переменные можно задать одним из следующих способов:

через файл terraform.tfvars (для этого скопируйте файл terraform.tfvars.example в terraform.tfvars и заполните необходимые значения);

с помощью флага -var при запуске команд Terraform;

через переменные окружения с префиксом TF_VAR_ (например, TF_VAR_cloud_id, TF_VAR_folder_id и т. д.).

## Порядок использования
Выполните следующие шаги для развёртывания инфраструктуры:
1. Скопируйте пример файла переменных и заполните его значениями:

```bash
cp terraform.tfvars.example terraform.tfvars
```
Инициализируйте провайдеры:
```bash
terraform init
```
Просмотрите план развёртывания:
```bash
terraform plan
```
Примените конфигурацию:
```bash
terraform apply
```
После успешного применения конфигурации будет выведен публичный IP‑адрес виртуальной машины (через  `output "ip" `).

## Подключение к виртуальной машине
Для подключения к ВМ по SSH используйте команду:
```bash
ssh ubuntu@<IP из output>
```
## Настройка Plex Media Server
Установка Plex Media Server выполняется автоматически при первом запуске виртуальной машины с помощью механизма cloud‑init:
- на инстанс записывается скрипт установки из репозитория;
- скрипт запускается один раз и выполняет развёртывание Plex.

## Основные параметры работы Plex
- Веб‑интерфейс: доступен по адресу http://<IP ВМ>:32400/web.
- Каталог для хранения медиаконтента: /media/movies. Владелец каталога — plex:plex, права доступа — 755.
- Лог установки: просмотреть лог можно командой sudo cat /var/log/plex-install.log.
- Версия Plex: задаётся в скрипте scripts/install-plex.sh через переменную PLEX_VERSION. Скрипт является идемпотентным — при повторном запуске установка не дублируется.

## Структура проекта
Проект имеет следующую структуру каталогов и файлов:

.
├── main.tf                      # Основной файл конфигурации: описание сети, подсети, ВМ, провайдера, переменных и output
├── cloud-init-plex.yaml.tpl     # Шаблон cloud-init: запись скрипта установки и его запуск
├── scripts/
│   └── install-plex.sh          # Скрипт установки Plex (идемпотентный, с логированием)
├── terraform.tfvars.example     # Пример файла переменных — скопировать в terraform.tfvars
├── terraform.tfvars             # Файл с вашими значениями (создаётся вручную, не должен попадать в репозиторий)
└── README.md
